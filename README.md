# 섯다 멀티플레이 웹 게임

SQLite + Express + Socket.IO 기반의 **2~6인용 섯다 웹 게임**입니다.

이 프로젝트는 아래 요구사항을 기준으로 만들었습니다.

- 로그인/회원가입 없이 **닉네임만 입력**해서 접속
- **SQLite(database.db)** 에 사용자 보유 재화 저장
- **Socket.IO Room** 으로 방 생성 / 방 참가 / 멀티플레이 처리
- 서버가 카드 분배, 족보 판정, 승패 계산, 조각 정산을 모두 처리하는 **권위적 서버 모델**
- 모바일 기준으로 보기 쉬운 **반응형 UI**
- **개발자 닉네임 + 관리자 비밀키**로 조각을 지급/차감할 수 있는 기능 포함

---

## 1. 이 프로젝트의 재화 구조

초기 명세의 DB 컬럼명은 `chips` 이지만, 실제 게임 화면에서는 전부 **조각**이라는 이름으로 표시됩니다.

즉, 아래 둘은 같은 뜻입니다.

- DB 컬럼 `users.chips`
- 화면에 보이는 `조각`

기본 지급량은 **1000조각**입니다.

## 3. 베팅 방식

- 매 게임 시작 시 모든 참가자에게 **참가비 1조각**이 자동 차감됩니다.
- 이후 한 번의 배팅 라운드가 진행됩니다.
- 플레이어는 자신의 차례에 아래 셋 중 하나를 선택합니다.
  - **콜 / 체크**
  - **베팅**: 텍스트박스에 입력한 수만큼 추가 베팅
  - **다이**
- 판돈(`pot`)은 서버가 관리합니다.

> 참고: 이 버전은 사이드팟/올인 시스템 없이 단순 구조로 작성되어 있습니다. 즉, 현재 필요한 금액을 감당할 수 없으면 다이를 선택하는 방식입니다.

## 4. 관리자(개발자) 조각 지급 기능

개발자용 조각 변경 기능이 포함되어 있습니다.

### 동작 조건
1. 접속 닉네임이 `DEV_NICK` 환경 변수와 같아야 합니다.
2. 클라이언트의 개발자 패널에서 입력한 비밀키가 `ADMIN_SECRET` 과 일치해야 합니다.

### 가능한 작업
- 양수 입력: 조각 지급
- 음수 입력: 조각 차감

예시:
- `100` 입력 → 100조각 지급
- `-50` 입력 → 50조각 차감

- 프로젝트가 만들어지면 편집기가 열립니다.

#### 3) 기본 파일 정리
기본으로 생성된 예제 파일이 있다면 아래 파일들을 기준으로 바꿔 주세요.

- `package.json`

- `server/server.js`

- `public/index.html`

#### 4) 폴더 생성
Glitch 편집기에서 아래 폴더를 만듭니다.

#### 5) 각 파일 내용 붙여넣기
이 프로젝트의 각 파일 내용을 복사해서 Glitch의 같은 이름 파일에 그대로 붙여넣습니다.

특히 `package.json`의 `start` 스크립트가 반드시 아래와 같아야 합니다.

#### 9) 친구와 같이 접속 테스트

## 8. 데이터베이스 구조

### users
```sql
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  chips INTEGER NOT NULL DEFAULT 1000
)
```

- `id`: 닉네임
- `chips`: 보유 조각 수

### game_history
```sql
CREATE TABLE game_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  winner_id TEXT NOT NULL,
  loser_id TEXT NOT NULL,
  bet_amount INTEGER NOT NULL,
  timestamp TEXT NOT NULL
)
```

- `winner_id`: 승자 닉네임
- `loser_id`: 패자 닉네임
- `bet_amount`: 기록된 이동 금액
- `timestamp`: ISO 문자열 시간

## 9. 실제 플레이 방법

### 방 만들기
1. 닉네임 입력
2. **방 만들기** 클릭
3. 화면 상단의 방 코드를 친구에게 알려줌

### 방 참가
1. 닉네임 입력
2. 방 코드 입력
3. **방 참가** 클릭

### 게임 시작
- 방장만 **게임 시작** 버튼을 누를 수 있습니다.
- 인원은 **2명 이상 6명 이하** 여야 합니다.
- 모든 참가자는 최소 1조각 이상 가지고 있어야 합니다.

### 배팅
- 자신의 차례에
  - 콜 / 체크
  - 원하는 만큼 베팅
  - 다이
  중 하나를 선택합니다.

## 11. 트러블슈팅

### Q1. sqlite3 설치 에러가 납니다.
Glitch/로컬 Node 버전이 너무 낮으면 문제가 생길 수 있습니다.
이 프로젝트는 `package.json`에 Node 20 이상을 권장하도록 적어 두었습니다.

### Q2. 같은 닉네임으로 두 번 접속이 안 됩니다.
중복 접속 방지를 위해 같은 닉네임은 동시에 1번만 접속되도록 막아 두었습니다.

### Q3. 게임 도중 창을 닫으면 어떻게 되나요?
해당 플레이어는 자동 다이 처리되고, 상황에 따라 남은 플레이어가 자동 승리할 수 있습니다.

### Q4. 조각이 부족하면 게임 시작이 안 되나요?
네. 참가비 1조각 이상이 있어야 게임 시작이 가능합니다.

## SQLite 영구 저장(배포용 권장)

이 프로젝트는 기본적으로 `database.db` 파일을 사용합니다.
배포 플랫폼에서 영구 저장 디스크/볼륨을 사용할 경우, 환경변수 `DB_PATH`를 예를 들어 `/data/database.db` 로 설정하면 재시작 후에도 데이터가 유지됩니다.
